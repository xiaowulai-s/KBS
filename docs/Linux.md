智能 Linux 门禁系统详细技术方案
1. 硬件资源深度规划 (Hardware Deep Dive)
基于 Linux 的灵活性，建议采用具备 NPU 加速的国产 SoC 方案。
1.1 核心计算平台
推荐芯片: Rockchip RK3568 (4核 A55, 1T NPU) 或 RK3588 (8核, 6T NPU，适合多路识别)。
内存: 2GB/4GB LPDDR4X（Linux 系统及人脸模型加载需要充足内存）。
存储: 16GB/32GB eMMC + MicroSD (用于本地日志与备份人脸库)。
1.2 感知与交互模块
双目视觉:
RGB 摄像头: MIPI-CSI 接口（低延迟），用于人脸特征提取。
IR 红外摄像头: 配合红外补光灯，用于活体检测及夜间识别。
射频模块: PN532 (支持 I2C/SPI/HSU)，兼容 ISO14443A/B，支持手机 NFC 模拟开门。
显示输出: 7~10寸 LVDS/MIPI 屏，运行 Wayland 或 X11 桌面环境。
音频: 采用 I2S 接口的 Audio Codec（如 WM8960），实现高保真语音播报及双向语音对讲。
1.3 工业级控制与扩展接口
门锁控制: 1路带光耦隔离的继电器输出。
自动门通讯: RS485（用于对接自动门机组的 Modbus 协议）或 CAN 总线（适合复杂工业场景）。
韦根接口 (Wiegand): 1路 Input/Output，确保兼容旧式第三方读卡器或门禁控制器。
报警输入: 2路数字输入（DI），用于对接消防报警和防拆开关。
2. Linux 软件架构 (Software Stack)
采用层次化设计，确保系统组件可独立升级。
2.1 操作系统层 (OS Layer)
发行版: 建议使用 Ubuntu Base 或 Yocto 定制化 Linux。
内核: 主线 Linux Kernel 5.10+，集成 V4L2（视频）、ALSA（音频）、GPIO-Libgpiod。
OTA 升级: 集成 RAUC 或 Mender，支持双分区（A/B）固件无损升级。
2.2 中间件与驱动层 (Middleware)
视频流处理: GStreamer 或 FFmpeg，利用硬件多媒体单元 (VPU) 进行编解码。
神经网络推理: RKNN-Toolkit（适配瑞芯微 NPU），将模型量化为 INT8 以提升性能。
数据库: SQLite3（本地存储）+ Redis（用于高频人脸特征缓存）。
2.3 应用功能层 (Application Layer)
系统采用多进程/多线程架构：
Main_Service (主控进程): 负责全局业务逻辑调度、状态机管理。
Face_Engine (识别进程): 负责 Camera 采集、人脸检测、活体比对。
UI_Process (交互进程): 基于 Qt 6 (PySide6/C++) 或 LVGL，负责图形化界面展示。
Comm_Agent (通信进程): 负责 MQTT/WebSocket 与云端心跳及数据同步。
3. 核心功能逻辑与算法 (Algorithm & Logic)
3.1 人脸识别全流程
预处理: 自动曝光调整 + 图像去噪。
检测 (Detection): 使用轻量级 RetinaFace 模型检测人脸位置。
对齐 (Alignment): 提取 5 点关键点，对人脸进行仿射变换，纠正姿态。
活体检测 (Anti-Spoofing): 比对 RGB 和 IR 帧，分析红外反射特征，过滤电子屏图片。
特征提取 (Extraction): 生成 128D 或 512D 特征向量。
搜索 (Searching): 本地毫秒级欧氏距离/余弦相似度比对。
3.2 自动门联动逻辑
软启软停: 通过 PWM 或 RS485 控制门机速度曲线，避免冲击。
智能感应: 识别成功后，根据人脸框大小判断人距门的距离，动态决定开启角度。
防夹逻辑:
一级: 视觉检测门缝区域是否有障碍物。
二级: 红外对射传感器物理中断。
三级: 电机电流监测（受阻电流激增即反转）。
4. 系统扩展性设计 (Extensibility)
为了让项目不仅仅是一个“单机门禁”，可以做以下扩展：
4.1 云端与 IoT 联动
远程下发: 通过 MQTT 协议从 Web 后台实时下发名单，无需现场操作。
微信/钉钉集成: 识别成功后，向用户手机推送“到岗/归家”通知。
访客系统: 支持手机生成的临时 QR Code (二维码) 扫描开门。
4.2 边缘计算与监控联动
RTSP 推流: 设备可作为一路网络摄像头 (IPC)，将实时画面推送给 NVR 录像机。
异常告警: 发现黑名单人员或长时间逗留，触发云端告警并抓拍上传。
4.3 语音对讲扩展
SIP 协议: 集成 Linphone SDK，支持访客呼叫住户室内机或手机 APP 进行远程通话及开门。
5. 安全性与可靠性 (Security & Reliability)
硬件守门狗 (Watchdog): 开启 Linux 内核看门狗，程序死机自动重启。
文件系统只读: 将系统关键分区设为 Read-Only，防止意外断电导致的系统崩溃。
加密芯片: 使用 ATECC608A 等加密芯片存储秘钥，防止人脸数据库被拷贝。
隐私模式: 支持本地识别，不上传原始图像，仅上传识别统计数据。
